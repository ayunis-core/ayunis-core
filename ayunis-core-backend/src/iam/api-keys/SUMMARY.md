API Keys
Org-scoped credentials for programmatic API access.

Manages creation, listing, and revocation of API keys that authenticate external programmatic access to Ayunis. Keys are scoped to an organization (not a user) and managed by org admins. The full plaintext key is shown only once at creation time; the database stores a bcrypt hash plus an indexed plaintext lookup prefix for fast lookup at request time.

Key entity is `ApiKey` (id, name, prefix, hash, expiresAt, revokedAt, orgId, createdByUserId, timestamps), exposing `KEY_PREFIX = 'ayk_live_'` and `LOOKUP_PREFIX_LENGTH = 12` as static constants so the use case, the DTO mapper, and the future request-time verifier all share one source of truth for the key format. Use cases include `CreateApiKeyUseCase` (generates a 256-bit random secret formatted as `ayk_live_<base64url>`, derives the lookup prefix, hashes the full secret via `HashTextUseCase`, persists with one retry on the rare prefix-collision case), `ListApiKeysByOrgUseCase` (returns keys for the caller's org, including revoked keys for audit-trail visibility), and `RevokeApiKeyUseCase` (soft-deletes a key via a conditional `UPDATE … SET revoked_at = NOW() WHERE id = $1 AND revoked_at IS NULL` so concurrent revokes preserve the original timestamp; surfaces `ApiKeyNotFoundError` for both truly-missing and cross-org cases so callers cannot enumerate IDs across organizations — cross-org attempts are still logged at warn level for incident review). Authorization is enforced at the controller via `@Roles(UserRole.ADMIN)`, and create/revoke routes are rate-limited via `@RateLimit`. Integrates with **hashing** for bcrypt operations and **orgs**/**users** for FK relationships (cascade-delete on org removal, set-null on creator removal to preserve audit).

The request-time auth strategy that validates incoming keys against the prefix index lands in a follow-up (iter 2 of AYC-92); the OpenAI-compatible chat completions endpoint follows in iter 3.
