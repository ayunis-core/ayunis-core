User Invitations
Manages organization member invitations with email and token workflow.

Handles the full invitation lifecycle for onboarding users into organizations. Supports creating single and bulk invites, sending invitation emails with JWT tokens, accepting invites, resending expired invites, deletion, and a scheduled TTL purge of expired invites.

This module manages how new users join existing organizations in Ayunis. The core entity is `Invite`, containing email, target org, assigned role (admin/user), inviter reference, expiration date, and acceptance timestamp. Key use cases include `CreateInviteUseCase` and `CreateBulkInvitesUseCase` for issuing invitations, `SendInvitationEmailUseCase` for dispatching email with a signed JWT link, `AcceptInviteUseCase` for processing token-based acceptance and creating the user account, `ResendExpiredInviteUseCase` for refreshing expired invitations, `GetInvitesByOrgUseCase` for listing pending invites, and `GetInviteByTokenUseCase` for token validation. It integrates with **users** for account creation upon acceptance, **orgs** for organization membership, and **authentication** for the invite JWT service. The `InvitesRepositoryPort` abstracts persistence with a local PostgreSQL implementation using TypeORM.

For data-privacy compliance, `PurgeExpiredInvitesUseCase` (driven by the nightly `PurgeExpiredInvitesTask` cron at 1 AM) hard-deletes **pending** invites whose `expiresAt` is older than a configurable grace period (`PURGE_INVITE_GRACE_DAYS`, default 30 days) so invitee email addresses are not retained indefinitely. Accepted invites are never purged (they are retained for referential integrity — see AYC-299). A `PURGE_DRY_RUN` flag makes the job report the impact without deleting.
