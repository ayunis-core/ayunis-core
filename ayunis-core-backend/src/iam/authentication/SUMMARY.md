User Authentication
Handles login, registration, JWT tokens, API-key authentication, and session management.

Manages user authentication flows including login with email/password, new user registration, JWT access/refresh token issuance, and current user retrieval. Uses Passport strategies with guards protecting routes globally.

This module is the entry point for all identity verification in Ayunis. Key entities include `AuthTokens` (access/refresh token pairs), `ActiveUser` (authenticated session representation with role, org, and email verification status), and `ActiveApiKey` (authenticated API key principal with org, role, and label). The discriminated-union type `ActivePrincipal` (`ActiveUser | ActiveApiKey`, tagged on `kind`) represents any authenticated caller and is what Passport assigns to `request.user`. Main use cases are login (credential validation and token generation), register-user (account creation with org provisioning), refresh-token (session extension), and get-current-user (profile retrieval). It integrates with the **users** module for credential storage and lookup, **hashing** for password verification, **orgs** for organization creation during registration, and **authorization** for role-based guard enforcement. The `UserContextInterceptor` populates `ContextService` with the authenticated principal's identifiers (userId or apiKeyId) and orgId for downstream use cases. A global `GlobalAuthGuard` (registered via `APP_GUARD`) secures all routes by default unless explicitly marked `@Public()`. The guard composes Passport strategies via `AuthGuard(['jwt', 'api-key'])`: the `jwt` strategy (defined here, plus a cookie-based refresh-token fallback) handles browser sessions, and the `api-key` strategy (defined in **api-keys**) handles `Authorization: Bearer ayk_live_...` callers. Both strategies populate `request.user` with an `ActivePrincipal`, so all global guards (subscription, IP allowlist, rate limit, etc.) apply uniformly regardless of credential type.
