Subscription Billing
Manages organization subscriptions, billing info, seats, credits, and cancellations.

Handles the complete subscription lifecycle for organizations including creation, seat/credit management, billing information, pricing, cancellation, and reactivation. Supports two subscription types: seat-based (with monthly/yearly renewal cycles and per-seat pricing) and usage-based (with monthly credits).

This module manages commercial access to Ayunis. Key entities use a polymorphic model: `Subscription` (base entity, org-scoped, with cancellation status and billing info), `SeatBasedSubscription` (extends `Subscription` with seat count, per-seat price, renewal cycle, and cycle anchor date), and `UsageBasedSubscription` (extends `Subscription` with monthly credits). The `SubscriptionType` enum differentiates between `SEAT_BASED` and `USAGE_BASED` subscriptions. Supporting entities include `SubscriptionBillingInfo` (company name, address, VAT number) and the `RenewalCycle` enum (monthly/yearly, for seat-based only). Main use cases include `CreateSubscriptionUseCase` (handles both subscription types), `CancelSubscriptionUseCase`, `UncancelSubscriptionUseCase` (reactivation before period ends), `UpdateSeatsUseCase` (adjust licensed user count for seat-based), `UpdateBillingInfoUseCase`, `GetActiveSubscriptionUseCase`, `HasActiveSubscriptionUseCase`, and `GetCurrentPriceUseCase`. Utility functions handle cycle date calculation and active-status determination. It integrates with **orgs** (each subscription belongs to one org), **authorization** via the `SubscriptionGuard` (gates features behind active subscriptions), **quotas** (subscription tier affects limits), and **trials** (organizations transition from trial to paid). The HTTP layer includes both user-facing and super-admin controllers.
