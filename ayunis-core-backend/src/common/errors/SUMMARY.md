Error Foundation
Base application error class and unauthorized access error definition.

The errors directory defines the `ApplicationError` abstract base class that all domain-specific errors extend. It provides error code, HTTP status code, metadata, and a `toHttpException()` method that maps to NestJS exceptions. The `UnauthorizedAccessError` is a concrete 403 error for authorization failures.

The `ApplicationError` base class is the cornerstone of the backend's error handling strategy. Every domain module defines its own error hierarchy extending this class (e.g., `AgentError`, `ThreadError`). Each error carries a string `code` for programmatic identification, a numeric `statusCode` for HTTP mapping, and optional `metadata` for additional context. The `toHttpException()` method converts domain errors to NestJS HTTP exceptions (`UnauthorizedException`, `ForbiddenException`, `NotFoundException`, `ConflictException`, `InternalServerErrorException`, `NotImplementedException`, `GatewayTimeoutException`, `BadRequestException`) based on status code, preserving code and metadata in the response body. The companion `ApplicationErrorFilter` in `../filters/` catches these errors globally, enriches Sentry reports with user context from the CLS store, sanitizes sensitive request data, and returns structured JSON error responses with timestamps and request paths. The `UnauthorizedAccessError` is a reusable 403 error used across modules when repository-level authorization checks fail.
