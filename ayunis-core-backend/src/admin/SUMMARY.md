Platform Administration
Super-admin API for managing AI models across all organizations.

The admin module provides platform-level administrative endpoints protected by a static token (`X-Admin-Token` header). It exposes CRUD operations for language and embedding models via the `AdminController`, bypassing JWT authentication entirely. Only super-admins with the correct token can access these routes.

The admin module implements super-admin functionality for platform-wide model management. The `AdminController` is decorated with `@UseGuards(AdminGuard)`, `@Public()` (bypasses JWT), and `@Admin()` (marks routes as admin-only). The `AdminGuard` reads the `X-Admin-Token` header and compares it against the configured `admin.adminToken` from environment variables, throwing `UnauthorizedException` on mismatch. The controller delegates to use cases from the `ModelsModule`: `GetModelUseCase`, `GetAllModelsUseCase`, `CreateLanguageModelUseCase`, `CreateEmbeddingModelUseCase`, `UpdateLanguageModelUseCase`, `UpdateEmbeddingModelUseCase`, and `DeleteModelUseCase`. Request validation is handled by dedicated DTOs (`CreateLanguageModelDto`, `CreateEmbeddingModelDto`, `UpdateLanguageModelDto`, `UpdateEmbeddingModelDto`). The module imports `ModelsModule` for model use cases and `ConfigModule` for the admin token configuration. It exports `AdminGuard` so other modules can reuse the same token-based protection pattern if needed.
