Configuration Registry
Centralized environment-based configuration factories for all backend services and integrations.

The config directory contains two categories of files:

**Environment Bootstrap** — `env.ts` and `env-register.js` handle dotenv loading before any other code runs. `env.ts` is imported as the first line in TypeScript entry points (`main.ts`, `cli/main.ts`, seed scripts). `env-register.js` is a CommonJS preload script for `-r` flag usage with ts-node/TypeORM CLI. Both load `.env.dev` (per-slot overrides from `./dev`) then `.env` (defaults), ensuring dev environment variables take precedence.

**NestJS registerAs Factories** — The `*.config.ts` files are NestJS `registerAs` configuration factories that map environment variables to typed configuration objects. Each file covers a specific concern: admin tokens, app settings, authentication/JWT, emails/SMTP, embeddings, evals/Langfuse, internet search/Brave, legal versioning, Locaboo, MCP URLs, model provider API keys, retrieval/Docling, storage/MinIO, subscriptions/pricing, tools limits, TypeORM/Postgres, and web/CORS.

The configuration module provides strongly-typed, namespaced configuration for every backend subsystem via NestJS `registerAs` factories. `app.config.ts` controls port, deployment mode (cloud/self-hosted), registration toggle, frontend URLs, and webhook endpoints. `authentication.config.ts` configures JWT secrets, token expiration for access/refresh/invite/email-confirmation/password-reset tokens, cookie settings, and auth provider selection (local vs cloud). `models.config.ts` holds API keys and base URLs for all LLM providers (Mistral, OpenAI, Anthropic, Bedrock, Ollama, Synaforce, Ayunis, OTC, Gemini, Azure). `typeorm.config.ts` builds PostgreSQL `DataSourceOptions` with environment-specific entity/migration paths and SSL settings for production. `storage.config.ts` defines MinIO connection parameters. `subscriptions.config.ts` sets per-seat pricing and trial message limits. `emails.config.ts` maps SMTP host/port/credentials. `legal.config.ts` tracks versioned terms-of-service and provider-specific legal acceptances. All configs are loaded globally in `AppModule` via `ConfigModule.forRoot()`.
