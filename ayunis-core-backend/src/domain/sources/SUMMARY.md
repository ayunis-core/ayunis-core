Data Sources
External data sources for RAG context enrichment

Sources manage user-uploaded files, URLs, and text content that feed the RAG pipeline. Each source is chunked, indexed, and made available for semantic search during AI conversations. Sources may optionally belong to a knowledge base via the `knowledgeBaseId` foreign key column.

The sources module manages external data that enriches AI context through RAG. The abstract `Source` entity specializes into `TextSource` (direct text with URL or content) and `DataSource` (uploaded files with storage references). `TextSourceContentChunk` represents indexed content fragments. Each source has an optional `knowledgeBaseId` foreign key linking it to a knowledge base collection; when a knowledge base is deleted the FK is set to `NULL` (`onDelete: SET NULL`). Key use cases include creating text and data sources, creating sources from uploaded files (`CreateSourcesFromFileUseCase` — detects file type, dispatches to the appropriate source creation use case, and handles document/CSV/spreadsheet files), creating processing sources (for async document pipelines), starting document processing (`StartDocumentProcessingUseCase` — orchestrates the full async pipeline by creating a PROCESSING source, uploading the file to MinIO, and enqueuing a BullMQ job; used by both the knowledge-bases and skills modules as the primary entry point for async document ingestion), marking sources as failed, querying text sources semantically, retrieving source details, and deleting sources (single or bulk). The async pipeline uses `DocumentProcessingPort` (application port) implemented by `DocumentProcessingProducer` (BullMQ adapter) to enqueue jobs; the port also exposes `cancelJob()` for best-effort cancellation of waiting or active jobs. The `DocumentProcessingConsumer` processes jobs asynchronously (download from MinIO → extract text → chunk → index → update status). `StaleProcessingCleanupService` runs a cron job every 5 minutes to mark sources stuck in PROCESSING for >15 minutes as FAILED. `SourceProcessingCleanupService` cancels an in-flight processing job via the port and removes associated MinIO processing files. The `SourceRepository` provides `findStaleProcessingSources()` for stale-source cleanup and `updateStatusConditionally()` for atomic status transitions guarded by an expected current status. Sources track their creator (user or system) and support assignment to both agents and threads. The module integrates with **rag** for content ingestion and vector indexing, **retrievers** for extracting text from files and URLs, **storage** for file persistence in MinIO, **agents** for source assignment to AI assistants, **threads** for per-conversation source attachment enabling contextual document queries, and **knowledge-bases** for grouping sources into named document collections.
