Data Sources
External data sources for RAG context enrichment

Sources manage user-uploaded files, URLs, and text content that feed the RAG pipeline. Each source is chunked, indexed, and made available for semantic search during AI conversations. Sources may optionally belong to a knowledge base via the `knowledgeBaseId` foreign key column.

The sources module manages external data that enriches AI context through RAG. The abstract `Source` entity specializes into `TextSource` (direct text with URL or content) and `DataSource` (uploaded files with storage references); `UrlSource` further specializes `TextSource` with a `maxDepth` property controlling how many levels of same-site links are crawled. `TextSourceContentChunk` represents indexed content fragments. Each source has an optional `knowledgeBaseId` foreign key linking it to a knowledge base collection; when a knowledge base is deleted its sources are removed with it (`onDelete: CASCADE`, see the AddOrgCascadeToConversationKbMcpData migration). Key use cases include creating text and data sources, creating sources from uploaded files (`CreateSourcesFromFileUseCase` — detects file type via the `FileType` enum (`PDF`, `DOCX`, `PPTX`, `TXT`, `AUDIO`), dispatches to the appropriate source creation use case, and handles document/CSV/spreadsheet/audio files), creating processing sources (for async document pipelines), creating processing URL sources and crawling them asynchronously (`CreateProcessingUrlSourceUseCase`, `EnqueueUrlCrawlUseCase`, `StartUrlCrawlUseCase`), marking sources as failed, querying text sources semantically, retrieving source details, deleting sources (single or bulk), and finding unreferenced source IDs (`FindUnreferencedSourceIdsUseCase` — accepts candidate IDs via `FindUnreferencedSourceIdsQuery` and returns the subset not attached to any skill, agent, or knowledge base, used by cross-module cleanup flows). The async pipeline uses `DocumentProcessingPort` (application port) implemented by `DocumentProcessingProducer` (BullMQ adapter) to enqueue jobs; the port also exposes `cancelJob()` for best-effort cancellation of waiting or active jobs. The `DocumentProcessingConsumer` processes jobs asynchronously (download from MinIO → extract text → chunk → index → update status). A parallel URL-crawl pipeline (`UrlCrawlModule`) uses `UrlCrawlProcessingPort` implemented by `UrlCrawlProducer` (BullMQ adapter, also exposing `cancelJob()`); the `UrlCrawlConsumer` crawls a root URL up to `maxDepth` levels of same-site links and indexes each page's chunks via the shared `SourceProcessingHelper` service (also used by `DocumentProcessingConsumer` for failure-marking and partial-index cleanup). `StaleProcessingCleanupService` runs a cron job every 5 minutes to mark sources stuck in PROCESSING for >15 minutes as FAILED. `SourceProcessingCleanupService` cancels in-flight jobs via both the document-processing and URL-crawl ports (best-effort, since it only knows the source id) and removes associated MinIO processing files; it resolves the org from the request context, so it only works pre-row-delete inside a request. `CleanupSourceProcessingUseCase` is its context-free counterpart for deferred deletion cleanup: it takes source ids plus an explicit org id, cancels both pipelines best-effort, and purges the `<orgId>/processing/<sourceId>/` prefixes via the storage module — usable after the owning rows are already cascade-deleted. The `SourceRepository` provides `findStaleProcessingSources()` for stale-source cleanup, `updateStatusConditionally()` for atomic status transitions guarded by an expected current status, and `findUnreferencedIds()` which returns the subset of candidate IDs that are older than a given threshold and not attached to any knowledge base, skill, or agent assignment. Sources track their creator (user or system) and support assignment to both agents and threads. The module integrates with **rag** for content ingestion and vector indexing, **retrievers** for extracting text from files and URLs, **storage** for file persistence in MinIO, **agents** for source assignment to AI assistants, **threads** for per-conversation source attachment enabling contextual document queries, and **knowledge-bases** for grouping sources into named document collections.
