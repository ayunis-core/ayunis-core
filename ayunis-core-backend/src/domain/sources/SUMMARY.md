Data Sources
External data sources for RAG context enrichment

Sources manage user-uploaded files, URLs, and text content that feed the RAG pipeline. Each source is chunked, indexed, and made available for semantic search during AI conversations. Sources may optionally belong to a knowledge base via the `knowledgeBaseId` foreign key column.

The sources module manages external data that enriches AI context through RAG. The abstract `Source` entity specializes into `TextSource` (direct text with URL or content) and `DataSource` (uploaded files with storage references). `TextSourceContentChunk` represents indexed content fragments. Each source has an optional `knowledgeBaseId` foreign key linking it to a knowledge base collection; when a knowledge base is deleted the FK is set to `NULL` (`onDelete: SET NULL`). Key use cases include creating text and data sources, creating sources from uploaded files (`CreateSourcesFromFileUseCase` — detects file type, dispatches to the appropriate source creation use case, and handles document/CSV/spreadsheet files), creating processing sources (for async document pipelines), marking sources as failed, querying text sources semantically, retrieving source details, deleting sources (single or bulk), and finding unreferenced source IDs (`FindUnreferencedSourceIdsUseCase` — accepts candidate IDs via `FindUnreferencedSourceIdsQuery` and returns the subset not attached to any skill, agent, or knowledge base, used by cross-module cleanup flows). The async pipeline uses `DocumentProcessingPort` (application port) implemented by `DocumentProcessingProducer` (BullMQ adapter) to enqueue jobs; the port also exposes `cancelJob()` for best-effort cancellation of waiting or active jobs. The `DocumentProcessingConsumer` processes jobs asynchronously (download from MinIO → extract text → chunk → index → update status). `StaleProcessingCleanupService` runs a cron job every 5 minutes to mark sources stuck in PROCESSING for >15 minutes as FAILED. `SourceProcessingCleanupService` cancels an in-flight processing job via the port and removes associated MinIO processing files. The `SourceRepository` provides `findStaleProcessingSources()` for stale-source cleanup, `updateStatusConditionally()` for atomic status transitions guarded by an expected current status, and `findUnreferencedIds()` which returns the subset of candidate IDs that are older than a given threshold and not attached to any knowledge base, skill, or agent assignment. Sources track their creator (user or system) and support assignment to both agents and threads. The module integrates with **rag** for content ingestion and vector indexing, **retrievers** for extracting text from files and URLs, **storage** for file persistence in MinIO, **agents** for source assignment to AI assistants, **threads** for per-conversation source attachment enabling contextual document queries, and **knowledge-bases** for grouping sources into named document collections.
