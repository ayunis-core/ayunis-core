Data Sources
External data sources for RAG context enrichment

Sources manage user-uploaded files, URLs, and text content that feed the RAG pipeline. Each source is chunked, indexed, and made available for semantic search during AI conversations. Sources may optionally belong to a knowledge base via the `knowledgeBaseId` foreign key column.

The sources module manages external data that enriches AI context through RAG. The abstract `Source` entity specializes into `TextSource` (metadata-only with a `textType` discriminator; subtypes `FileSource` and `UrlSource`) and `DataSource` (uploaded files with storage references). `TextSource` does not hold inline text or content chunks—text and chunks are passed separately through the repository's `saveTextSource` method and retrieved via dedicated query use cases. `TextSourceContentChunk` represents indexed content fragments. Each source has an optional `knowledgeBaseId` foreign key linking it to a knowledge base collection; when a knowledge base is deleted the FK is set to `NULL` (`onDelete: SET NULL`). Key use cases include creating text and data sources, querying text sources semantically, retrieving source details, retrieving sources by knowledge base ID (`GetSourcesByKnowledgeBaseIdUseCase`), finding content chunks by IDs (`FindContentChunksByIdsUseCase`), extracting text lines from a source (`ExtractTextLinesUseCase`), and deleting sources (single or bulk). The `SourceRepository` port exposes `findById`, `findByIds`, `findByKnowledgeBaseId`, `saveTextSource`, `save`, `extractTextLines`, `findContentChunksByIds`, `delete`, and `deleteMany` methods. Sources track their creator (user or system) and support assignment to both agents and threads. The module integrates with **rag** for content ingestion and vector indexing, **retrievers** for extracting text from files and URLs, **storage** for file persistence in MinIO, **agents** for source assignment to AI assistants, **threads** for per-conversation source attachment enabling contextual document queries, and **knowledge-bases** for grouping sources into named document collections.
